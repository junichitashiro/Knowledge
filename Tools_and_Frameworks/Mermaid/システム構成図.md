# システム構成図

---

## 基本システム構成テンプレ（3層アーキテクチャ）

* 典型的な Web / App / DB の3層構造

```mermaid
flowchart LR

User[ユーザー] --> LB[Load Balancer]

subgraph Web層
  WEB1[Web Server 1]
  WEB2[Web Server 2]
end

subgraph App層
  APP1[Application Server 1]
  APP2[Application Server 2]
end

subgraph DB層
  DB[(Database)]
end

LB --> WEB1
LB --> WEB2

WEB1 --> APP1
WEB2 --> APP2

APP1 --> DB
APP2 --> DB
```

---

## クラウド構成テンプレ（AWS系）

* WAF, VPC, ,Subnet, RDS などをサブグラフで表現


```mermaid
flowchart LR

User --> Internet

subgraph Cloud
  Internet --> WAF
  WAF --> ALB

  subgraph VPC
    subgraph PublicSubnet
      ALB
    end

    subgraph PrivateSubnet
      EC2_1[App Server]
      EC2_2[App Server]
    end

    subgraph DBSubnet
      RDS[(RDS Database)]
    end
  end
end

ALB --> EC2_1
ALB --> EC2_2

EC2_1 --> RDS
EC2_2 --> RDS
```

---

## マイクロサービス構成テンプレ

特徴
* API Gateway
* サービス分離
* DB分離

```mermaid
flowchart LR

User --> API[API Gateway]

subgraph Microservices
  Auth[Auth Service]
  Order[Order Service]
  Payment[Payment Service]
end

subgraph Databases
  UserDB[(User DB)]
  OrderDB[(Order DB)]
end

API --> Auth
API --> Order
API --> Payment

Auth --> UserDB
Order --> OrderDB
Payment --> OrderDB
```


---

## データパイプライン（ETL）

用途
* データ分析
* データ基盤
* AI学習基盤

```mermaid
flowchart LR

Source[外部データ] --> ETL[ETL処理]

subgraph DataPlatform
  DataLake[(Data Lake)]
  Warehouse[(Data Warehouse)]
end

ETL --> DataLake
DataLake --> Warehouse

Warehouse --> BI[BIツール]
```
