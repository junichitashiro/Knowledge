# フローチャート

---

## 基本構文

```mermaid
flowchart TD
A[開始] --> B[処理]
B --> C[終了]
```

- `TD` = Top Down（上→下）
- `LR` = Left Right（左→右）

---

## ノード形状

```mermaid
flowchart TD
A[四角]
B(丸)
C((二重丸))
D{条件}
E[/入出力/]
F[[サブルーチン]]
```

| 記法    | 意味         |
| ------- | ------------ |
| `[ ]`   | 処理         |
| `( )`   | 開始・終了   |
| `(( ))` | 強調         |
| `{ }`   | 条件分岐     |
| `[/ /]` | 入力出力     |
| `[[ ]]` | サブルーチン |

---

## 条件分岐

```mermaid
flowchart TD
A[開始] --> B{ログイン成功?}

B -- Yes --> C[ダッシュボード]
B -- No --> D[エラー表示]

C --> E[終了]
D --> E
```

---

## ループ

```mermaid
flowchart TD
A[開始] --> B{条件}
B -- Yes --> C[処理]
C --> B
B -- No --> D[終了]
```

---

## ノードIDと表示名

```mermaid
flowchart TD
A[Start]
B[Process]
C[End]

A --> B --> C
```

- `A B C` は内部ID
- `[Start]` は表示名

---

## 矢印

```mermaid
flowchart LR
A --> B
A -.-> C
A ==> D
```

| 記号   | 意味 |
| ------ | ---- |
| `-->`  | 通常 |
| `-.->` | 点線 |
| `==>`  | 強調 |

---

## 矢印ラベル

```mermaid
flowchart TD
A -->|成功| B
A -->|失敗| C
```

---

## サブグラフ

業務フロー整理に便利。

```mermaid
flowchart LR

subgraph 営業
A[訪問]
B[提案]
end

subgraph 受注
C[見積]
D[受注]
end

A --> B --> C --> D
```

---

## シーケンス図

APIや処理フロー説明に便利。

```mermaid
sequenceDiagram
participant User
participant System
participant DB

User ->> System: ログイン
System ->> DB: ユーザ確認
DB -->> System: OK
System -->> User: ログイン成功
```

| 記号   | 意味       |
| ------ | ---------- |
| `->>`  | 同期       |
| `-->>` | レスポンス |

## 実務でよくある業務フロー例

```mermaid
flowchart TD

A[問い合わせ] --> B{既存顧客?}

B -->|Yes| C[営業対応]
B -->|No| D[新規登録]

C --> E[見積作成]
D --> E

E --> F{受注?}

F -->|Yes| G[受注登録]
F -->|No| H[終了]

G --> I[納品]
```
